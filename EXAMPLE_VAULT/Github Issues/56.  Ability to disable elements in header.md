To disable `title` and `subtitle` - just don't add these properties in `trackerData` object.

In release 1.19.1 added new `ui` property in `trackerData` schema to disable elements in header:
- `hideTabs`
- `hideYear`
- `defaultView`
- `hideTitle`
- `hideSubtitle`

```dataviewjs
const trackerData = {
    intensityScaleStart: -2,
    intensityScaleEnd: 2,
    separateMonths: true,
    heatmapTitle: 'This title will be hidden',
    heatmapSubtitle: 'This subtitle will be hidden',
    ui: {
        hideTabs: true,
        hideYear: true,
        hideTitle: true,
        hideSubtitle: true,
        defaultView: 'heatmap-tracker', // heatmap-tracker | heatmap-tracker-statistics | documentation | legend
    },
    entries: [
    {
      "date": "2025-04-14",
      "intensity": -2
    },
    {
      "date": "2025-04-15",
      "intensity": -1
    },
    {
      "date": "2025-04-16",
      "intensity": 0
    },
    {
      "date": "2025-04-17",
      "intensity": 1
    },
     {
      "date": "2025-04-18",
      "intensity": 2
    },
    ]
}

renderHeatmapTracker(this.container, trackerData)

```

Using `defaultView` you can change the view which should be display, by default. For example `Statistics` view:
```dataviewjs

    intensityScaleStart: -2,
    intensityScaleEnd: 2,
    separateMonths: true,
    heatmapTitle: 'This title will be hidden',
    heatmapSubtitle: 'This subtitle will be hidden',
    ui: {
        hideTabs: true,
        hideYear: true,
        hideTitle: true,
        hideSubtitle: true,
        defaultView: 'heatmap-tracker-statistics', // heatmap-tracker | heatmap-tracker-statistics | documentation | legend
    },
    entries: [
    {
      "date": "2025-04-14",
      "intensity": -2
    },
    {
      "date": "2025-04-15",
      "intensity": -1
    },

      "intensity": 0

      "intensity": 1

     {

renderHeatmapTracker(this.container, trackerData)
```
